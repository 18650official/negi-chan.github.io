动手打造 Hexo 博客文章朗读功能

在搭建我的个人博客时，我一直有一个想法：让博客文章能够被朗读出来，给读者带来更好的沉浸式体验，也为可能的视力障碍读者提供无障碍辅助。
经过一番摸索和尝试，我终于成功实现了这个功能，并且在前端实现了播放、暂停和停止的完美控制。在这篇文章中，我将分享我的完整实践过程，希望能为你提供一些帮助。

项目背景与技术选型

我的博客基于 Hexo 框架，并使用了热门的 Butterfly 主题。起初，我尝试使用一些现有的插件，但它们大多功能有限或配置复杂。最终，我决定放弃插件，直接使用 微软 Azure 认知服务 提供的语音服务 API，通过前端代码直接调用，来获取和播放语音。

这样做的好处是：

  • 灵活性高：可以完全自定义播放器的样式和行为。
  • 无需后端：作为静态博客，所有逻辑都在前端完成。

整个项目主要涉及以下技术：

  • Pug：Hexo 主题的模板引擎，用于构建 HTML 结构。
  • Stylus：CSS 预处理器，用于美化界面。
  • JavaScript：核心逻辑，用于控制语音朗读和播放器状态。
  • Azure 语音服务：提供强大的 Text-to-Speech (TTS) API。

项目实现

第一步：API 密钥与服务准备

在开始编写代码之前，你需要在 Azure 门户 上创建一个语音服务（Speech Service）实例。

  1. 登录 Azure 账户，创建一个新的 “语音服务” 资源。
  2. 在资源创建完成后，进入该资源的 “密钥和终结点” 页面。
  3. 找到你的 API 密钥 (SPEECH_KEY) 和 区域 (SPEECH_REGION)，并妥善保存。这是我们后续代码中用于认证的关键信息。

  • API 请求的细节：特别是 400 Bad Request 错误，它提醒我需要处理 SSML 中的特殊字符。
